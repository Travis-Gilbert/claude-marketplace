# PATTERNS-promotion.md

How the captured -> reviewed -> promoted workflow governs epistemic status transitions.

## Build Sequence

### Step 1: Understand the Three Tiers

| Tier | Source | Trust Level | Example |
|------|--------|-------------|---------|
| Captured | Auto-generated by engine | Unreviewed, may be wrong | Auto-objectified PERSON, proposed Claim |
| Reviewed | Human has seen and confirmed | Validated but not canonical | User confirms a Claim is accurate |
| Promoted | Human explicitly promotes | Canonical, trusted knowledge | Claim becomes "supported", Method becomes "trusted" |

Invariant 7: **LLMs propose. Humans review. Nothing auto-promotes to canon.**

### Step 2: Auto-Generated Records

The engine creates records with initial status markers:

```python
# Claims from decomposition
Claim.objects.create(status='proposed', ...)

# Edges from engine passes
Edge.objects.create(is_auto=True, ...)

# Objects from auto-objectification
Object.objects.create(capture_method='auto', status='active', ...)

# Methods from LLM draft
Method.objects.create(status='draft', runtime_kind='llm_draft', ...)
```

### Step 3: Review Queues

Surface unreviewed items for human attention:

```python
def get_review_queue(notebook=None):
    """Return items needing human review, ordered by priority."""
    queue = []

    # Proposed claims with high contradiction scores
    contested = Claim.objects.filter(status='proposed').order_by('-created_at')
    queue.extend([('claim', c) for c in contested[:20]])

    # Auto-generated objects with low confidence
    auto_objects = Object.objects.filter(
        capture_method='auto', is_deleted=False
    ).order_by('-created_at')
    queue.extend([('object', o) for o in auto_objects[:20]])

    # Draft methods awaiting review
    draft_methods = Method.objects.filter(status='draft').order_by('-created_at')
    queue.extend([('method', m) for m in draft_methods[:10]])

    # Open tensions requiring resolution
    tensions = Tension.objects.filter(status='open').order_by('-confidence')
    queue.extend([('tension', t) for t in tensions[:10]])

    return queue
```

### Step 4: Status Transitions

Each epistemic primitive has its own status lifecycle:

```python
# Claim: proposed -> supported -> contested -> superseded -> archived
def promote_claim(claim, reviewer_note=''):
    claim.status = 'supported'
    claim.save(update_fields=['status', 'updated_at'])
    _record_promotion_event(claim, 'claim_promoted', reviewer_note)

# Method: draft -> candidate -> reviewed -> trusted -> deprecated -> archived
def promote_method(method, reviewer_note=''):
    VALID_TRANSITIONS = {
        'draft': 'candidate',
        'candidate': 'reviewed',
        'reviewed': 'trusted',
    }
    next_status = VALID_TRANSITIONS.get(method.status)
    if not next_status:
        raise ValueError(f'Cannot promote from {method.status}')
    method.status = next_status
    method.save(update_fields=['status', 'updated_at'])
    _record_promotion_event(method, 'method_promoted', reviewer_note)

# Edge: confirm auto-generated edge
def confirm_edge(edge, reviewer_note=''):
    edge.is_auto = False  # Now human-confirmed
    edge.save(update_fields=['is_auto', 'updated_at'])
    _record_promotion_event(edge, 'edge_confirmed', reviewer_note)
```

### Step 5: Timeline Events

Every promotion creates an immutable Timeline Node:

```python
def _record_promotion_event(obj, event_type, note=''):
    from .models import Node, Timeline
    timeline = Timeline.objects.filter(is_master=True).first()
    Node.objects.create(
        node_type='status_change',
        title=f'Promoted: {str(obj)[:80]}',
        body=note,
        timeline=timeline,
    )
```

### Step 6: Feedback Loop Integration

Promotions feed back into the learned scorer (PATTERNS-learned-scorer.md):

```python
# In signals.py
@receiver(post_save, sender=Edge)
def on_edge_confirmed(sender, instance, **kwargs):
    if not instance.is_auto and kwargs.get('update_fields') and 'is_auto' in kwargs['update_fields']:
        # Human confirmed this edge -> positive feedback signal
        ConnectionFeedback.objects.create(
            from_object=instance.from_object,
            to_object=instance.to_object,
            label='manual',
            label_strength=1.0,
        )
```

## Critical Constraints

- Nothing auto-promotes to canon -- a human must explicitly confirm
- Status transitions must follow the defined lifecycle (no skipping steps)
- Every promotion is recorded as an immutable Timeline Node
- Auto-generated records are always distinguishable (capture_method='auto', is_auto=True)
- Review queues prioritize high-confidence tensions and recent auto-generated items
- Promotions generate ConnectionFeedback for the learned scorer
- Bulk promotion is allowed but each item gets its own Timeline Node
