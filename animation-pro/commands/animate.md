Add purposeful animation to the specified component.

1. Load the motion-architect agent.
2. Run the Purpose Test on the requested animation.
3. If the test fails, report why and do not proceed.
4. Detect the existing animation stack.
5. Select the right tool (CSS, Motion, react-spring, etc.).
6. Load the relevant skill's SKILL.md and reference documents.
7. Route to the appropriate specialist agent.
8. Implement with spring physics where appropriate.
9. Implement prefers-reduced-motion (non-negotiable).
10. Verify only transform and opacity are animated.
11. Run the project's formatter and linter.

Report: what motion was added, why it passes the Purpose Test, which
tool was selected and why, and confirm reduced-motion compliance.
