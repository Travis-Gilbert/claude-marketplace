## Appendix

### Methodology

This audit was conducted using a combination of automated scanning tools, manual keyboard testing, and assistive technology testing. Each page in scope was evaluated against all Level A and AA success criteria in WCAG 2.2.

**Testing approach:**
1. Automated scan with [tool name] to identify programmatic issues
2. Manual keyboard navigation of all interactive elements
3. Screen reader testing with [screen reader name] on [OS]
4. Visual inspection for color contrast, text spacing, and reflow
5. Review of page structure (headings, landmarks, ARIA usage)

### Severity Definitions

| Severity | Definition |
|----------|------------|
| **Critical** | Completely blocks access for one or more user groups. The feature or content is unusable. |
| **Major** | Creates a significant barrier. Users may be able to work around it, but with considerable difficulty. |
| **Minor** | Creates an inconvenience but does not significantly block access. Users can accomplish their goal. |
| **Advisory** | Not a WCAG failure but a best practice recommendation that would improve the experience. |
