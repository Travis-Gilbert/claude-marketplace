# UX-Pro Agents

## Routing

Each agent is a specialist in one UX discipline. Use the slash commands
for direct access. For cross-disciplinary questions, the relevant agents
can be combined.

## Agent Index

| Agent | Domain | Slash Command |
|-------|--------|--------------|
| ux-researcher | Research planning, method selection, metrics | /ux-research |
| interaction-designer | Flow design, patterns, micro-interactions, states | /interaction |
| information-architect | Navigation, taxonomy, labeling, findability | /ia |
| accessibility-auditor | WCAG, ARIA, keyboard, inclusive design | /a11y |
| ux-writer | Microcopy, error messages, voice and tone | /ux-write |
| usability-tester | Test scripts, facilitation, analysis, reporting | /usability |
| service-designer | Blueprints, journey maps, experience mapping | /service |

## Cross-Disciplinary Patterns

| Task | Primary Agent | Supporting Agent |
|------|--------------|-----------------|
| Evaluating a new feature | interaction-designer | accessibility-auditor |
| Planning a research study | ux-researcher | usability-tester |
| Designing navigation | information-architect | interaction-designer |
| Writing error messages | ux-writer | accessibility-auditor |
| Mapping a service | service-designer | ux-researcher |
| Auditing an existing page | accessibility-auditor | interaction-designer |
| Reviewing UI copy | ux-writer | usability-tester |
