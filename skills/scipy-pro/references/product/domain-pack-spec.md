# Domain Pack Spec

## Objective
Keep one epistemic kernel while enabling domain-specific extraction and method behavior.

## Initial Packs
- `computer_science`
- `built_environment`
- `investigative`
- `civic_policy`

## Pack Components
- Parser hints and source typing preferences.
- Claim/entity/rule extraction templates.
- Promotion-review heuristics.
- Method DSL operator allow-list.
- Evaluation datasets and scoring criteria.

## Rollout Guidance
1. Ship one pack end-to-end first.
2. Prove promotion -> method -> run -> feedback loop.
3. Generalize shared abstractions into kernel.
4. Add next pack without branching kernel semantics.
